  1. Android异步批量下载图片并缓存

    周二 29 七月 2014   Write By Cow

    前言

    接触android开发不久，近段时间需实现一个批量下载图片并显示的小功能。在网上搜索了一圈，发现国内外网上异步加载的例子太多太杂，要么是加载大图decode时报OOM异常，要么内存急剧上升不稳定。在前辈们的基础上，做了一些优化，特共享出来，欢迎大家指正。这里主要参见了以下两篇文章，非常感谢：

    Android照片墙应用实现，再多的图片也不怕崩溃

    Android 异步加载图片，使用LruCache和SD卡或手机缓存，效果非常的流畅

    在巨人的肩膀上，主要优化了以下几个地方：

    • 下载大图decode时，可根据View大小自动缩放图片，不在出现OOM和SkImageDecoder::Factory returned null错误
    • 图片下载失败时，可自定义失败重试次数
    • 记录正在下载的任务，防止屏幕滚动时多次下载
    • 缓存目录容量大于给定限制时，清空文件缓存
    • 其他一些小问题

  4. 详解android:layout_weight

    周五 28 三月 2014   Write By Cow

    android:layout_weight

    android：weight是线性布局LinearLayout中非常重要的概念，在创造兼容性APP中有不可替代的功劳。它主要作用为分配剩余空间，剩余空间是在线性布局中把子组件所占空间分配完毕后所剩余的部分。每个子组件占剩余空间的比例等于自身所设置参数除以所有子组件所设置参数之和。
